# PolStan Technical Documentation

## Overview

PolStan is a mobile-first Next.js 15 application deployed as a Dockerized standalone server. The same web application is the source of truth for the public website, installable PWA and Android Trusted Web Activity.

Primary production URL:

- `https://polstan.ru/ru`
- `https://polstan.ru/en`

Android identity:

- Package: `ru.polstan.app`
- App name: `PolStan`
- Launch URL: `https://polstan.ru/ru`
- Wrapper: Trusted Web Activity generated by Bubblewrap

## Application Architecture

- `app/` contains App Router routes, metadata, sitemap, robots and the dynamic Digital Asset Links route.
- `components/polstan-app.tsx` is the main interactive client surface: hero video, services, drops, selection state and Telegram lead flow.
- `lib/content.ts` and `lib/types.ts` define localized copy and content schema.
- `lib/telegram.ts` builds the final Telegram deep link and message body.
- `public/manifest.webmanifest`, `public/sw.js` and `public/offline.html` provide installability and offline behavior.
- `android-twa/` contains the generated Android TWA project and release instructions.

The app intentionally keeps lead submission client-side. The form builds a Telegram URL with the selected offerings, budget, timeline, user contact and source URL. There is no server-side lead database in v1.0.0.

## Environment

Runtime variables:

- `NEXT_PUBLIC_TELEGRAM_USERNAME`: Telegram username without `@`; defaults to `polstan`.
- `ACME_EMAIL`: email used by Caddy for Let's Encrypt in the Caddy-only stack.
- `ANDROID_SHA256_CERT_FINGERPRINTS`: comma or whitespace separated Play App Signing SHA-256 fingerprints for Digital Asset Links.
- `POLSTAN_APP_BIND`: nginx-stack bind address; defaults to `127.0.0.1`.
- `POLSTAN_APP_PORT`: nginx-stack host port; defaults to `3002`.

Do not commit `.env`, keystores, APKs, AABs, local Android properties or `.secrets/`.

## PWA Behavior

The PWA manifest uses:

- `id`: `/ru`
- `start_url`: `/ru`
- `scope`: `/`
- `display`: `standalone`
- `orientation`: `portrait`
- `theme_color` and `background_color`: `#030405`

The service worker:

- precaches `/ru`, `/en`, `offline.html`, manifest and core icons;
- uses network-first behavior for navigations;
- uses cache-first behavior for Next static chunks;
- uses stale-while-revalidate for icons, images and manifest;
- avoids intercepting ranged media/video requests.

## Android TWA

The Android project lives in `android-twa/` and is generated from `https://polstan.ru/manifest.webmanifest`.

Key release settings:

- `applicationId`: `ru.polstan.app`
- `versionName`: `1.0.0`
- `versionCode`: `1`
- `compileSdkVersion`: `36`
- `targetSdkVersion`: `35`
- `minSdkVersion`: `23`
- fallback strategy: `customtabs`
- notification delegation: disabled

Digital Asset Links are served by `app/.well-known/assetlinks.json/route.ts`. The endpoint returns an empty array until `ANDROID_SHA256_CERT_FINGERPRINTS` is configured with the Play App Signing certificate fingerprint.

## Production Deployment

Current production uses the nginx VPS path:

```bash
docker compose -f docker-compose.nginx.yml -p polstan up -d --build
```

The app container binds to `127.0.0.1:3002` by default, and nginx proxies `polstan.ru` traffic to that port.

Expected verification after deployment:

```bash
curl -I https://polstan.ru/ru
curl -I https://polstan.ru/sw.js
curl https://polstan.ru/.well-known/assetlinks.json
docker compose -f docker-compose.nginx.yml -p polstan ps
```

If `ANDROID_SHA256_CERT_FINGERPRINTS` is not configured, `assetlinks.json` should return `[]`. After the Play App Signing fingerprint is configured, it should return one statement for package `ru.polstan.app`.

## Release Checklist

Before tagging a release:

1. Run `npm run lint`.
2. Run `npm run build`.
3. Run `npx --yes @bubblewrap/cli@1.24.1 doctor`.
4. Run `android-twa/gradlew.bat :app:assembleRelease :app:bundleRelease --no-daemon`.
5. Smoke test production mobile route and `/.well-known/assetlinks.json`.
6. Confirm `.env`, keystores, APKs and AABs are not staged.
7. Tag the release and push the tag.

## Known v1.0.0 Follow-ups

- Add the real Play App Signing SHA-256 fingerprint to production after the first internal testing upload.
- Run Play Console internal testing on physical Android devices.
- Create Play Store listing screenshots from the verified mobile UI.
